---
title: "PR lifecycle"
description: "How pull request automation relates to Kata CLI."
---

`@kata-sh/cli` 0.16.0 preferences configure backend settings for Kata workflow operations.

PR lifecycle automation is handled by Kata Symphony and the active agent harness. Use Kata CLI for setup, backend diagnostics, and typed workflow operations such as `issue.get`, `task.updateStatus`, `artifact.write`, and `health.check`.

For Symphony ticket-to-merge behavior, see [Kata Symphony](/symphony/overview).
