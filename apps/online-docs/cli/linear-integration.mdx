---
title: "Linear integration"
description: "Configure Kata CLI for Linear-backed runtime operations."
---

Kata CLI 0.16.0 supports Linear as a runtime backend for project, milestone, slice, task, issue, status, artifact, and health operations.

## Setup

Run setup with the Linear backend:

```bash
kata setup --yes \
  --backend linear \
  --linear-workspace <workspace> \
  --linear-team <team> \
  --linear-project <project>
```

Use `--linear-auth-env <VAR>` when your token is stored in a custom environment variable.

## Preferences

Setup writes `.kata/preferences.md` with the required Linear fields:

```yaml
---
workflow:
  mode: linear
linear:
  workspace: <workspace>
  team: <team>
  project: <project>
  authEnv: LINEAR_API_KEY
  states:
    backlog: Backlog
    todo: Todo
    in_progress: In Progress
    agent_review: Agent Review
    human_review: Human Review
    merging: Merging
    done: Done
---
```

Required fields are `linear.workspace`, `linear.team`, and `linear.project`.

Optional fields include `linear.authEnv`, `linear.activeMilestoneId`, `linear.states`, and `linear.labels`.

## Authentication

Kata resolves Linear auth from:

1. the environment variable named by `linear.authEnv`
2. `LINEAR_API_KEY`
3. `LINEAR_TOKEN`

```bash
export LINEAR_API_KEY=lin_api_...
```

## Doctor checks

Run doctor after setup:

```bash
kata doctor
kata doctor --json
```

For Linear mode, doctor checks auth, project access, workflow states, and GraphQL capabilities.
