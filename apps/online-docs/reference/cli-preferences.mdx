---
title: "CLI preferences"
description: "Reference for backend preferences in .kata/preferences.md."
---

`kata` reads backend settings from `.kata/preferences.md` in the current workspace.

## GitHub mode example

```yaml
---
workflow:
  mode: github
github:
  repoOwner: kata-sh
  repoName: kata-mono
  stateMode: projects_v2
  githubProjectNumber: 12
---
```

## Linear mode example

```yaml
---
workflow:
  mode: linear
linear:
  workspace: <workspace>
  team: <team>
  project: <project>
  authEnv: LINEAR_API_KEY
  states:
    backlog: Backlog
    todo: Todo
    in_progress: In Progress
    agent_review: Agent Review
    human_review: Human Review
    merging: Merging
    done: Done
---
```

## Required fields

### `workflow.mode`

- Type: `string`
- Allowed values: `github`, `linear`

### `github.repoOwner`

- Type: `string`
- Required when `workflow.mode: github`

### `github.repoName`

- Type: `string`
- Required when `workflow.mode: github`

### `github.stateMode`

- Type: `string`
- Required value: `projects_v2`
- Required when `workflow.mode: github`

### `github.githubProjectNumber`

- Type: `number`
- Required when `workflow.mode: github`
- Must be a positive integer

### `linear.workspace`

- Type: `string`
- Required when `workflow.mode: linear`

### `linear.team`

- Type: `string`
- Required when `workflow.mode: linear`

### `linear.project`

- Type: `string`
- Required when `workflow.mode: linear`

## Optional Linear fields

### `linear.authEnv`

- Type: `string`
- Named environment variable to read for the Linear API key; overrides the default `LINEAR_API_KEY` and `LINEAR_TOKEN` lookup
- Default written by setup: `LINEAR_API_KEY`

### `linear.activeMilestoneId`

- Type: `string`
- Optional active milestone identifier override

### `linear.states`

- Type: `object`
- Optional mapping from Kata lifecycle state keys to Linear workflow state names

### `linear.labels`

- Type: `object`
- Optional label mapping used by Linear backend operations

## Setup behavior

`kata setup` provisions GitHub mode by default and Linear mode when `--backend linear` is selected. It writes `.kata/preferences.md` when the file does not exist.

```bash
kata setup --yes --backend github --repo owner/name --project-number 12
kata setup --yes --backend linear --linear-workspace <workspace> --linear-team <team> --linear-project <project>
```

## Validation behavior

Use `kata doctor` to validate:

- skills installation
- backend config parsing
- GitHub auth and GitHub Projects v2 fields when mode is `github`
- Linear auth, project access, workflow states, and GraphQL capabilities when mode is `linear`
