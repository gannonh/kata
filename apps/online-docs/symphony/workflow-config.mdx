`WORKFLOW.md` has two parts:

1. YAML front-matter: orchestrator configuration
2. Markdown body: Liquid prompt template (unless per-state `prompts` is configured)

Symphony hot-reloads config updates from this file.

## Current worker/helper flow

Use `agent.*` to define how Symphony launches the worker runtime, and use the sym-state helper for tracker/document/state operations from worker prompts/skills:

```bash
.agents/skills/sym-state/scripts/sym-call <operation> --input /tmp/input.json
```

Operations:

- `issue.get`
- `issue.list-children`
- `document.read`
- `document.write`
- `comment.upsert`
- `issue.update-state`
- `issue.create-followup`

Legacy `kata_*` worker tool names are not the current contract.

## Minimal config

```yaml
---
tracker:
  kind: linear
  api_key: $LINEAR_API_KEY
  project_slug: your-project
  active_states:
    - Todo
    - In Progress
    - Agent Review
    - Merging
    - Rework
  terminal_states:
    - Closed
    - Cancelled
    - Canceled
    - Duplicate
    - Done

workspace:
  root: ~/symphony-workspaces
  repo: /absolute/path/to/repo
  git_strategy: worktree
  isolation: local
  branch_prefix: symphony
  clone_branch: main
  base_branch: main

agent:
  name: pi
  command: pi --mode rpc
  model: anthropic/claude-opus-4-6
  model_by_state:
    Agent Review: anthropic/claude-sonnet-4-6
    Merging: anthropic/claude-sonnet-4-6
  no_session: true
  max_concurrent_agents: 1
  max_turns: 20
  approval_policy: never
  thread_sandbox: danger-full-access
---
```

## Section reference

### `tracker`

- `kind`: `linear` or `github`
- `api_key`: token (`$VAR` supported)
- Linear fields: `project_slug`, optional `workspace_slug`
- GitHub fields: `repo_owner`, `repo_name`, optional `github_project_number`
- dispatch filters: `active_states`, `terminal_states`, `exclude_labels`

### `workspace`

- `root`, `repo`, `branch_prefix`, `clone_branch`, `base_branch`
- `git_strategy`: `auto`, `clone-local`, `clone-remote`, `worktree`
- `isolation`: `local` or `docker`

Worktree bootstrap behavior:

- requires local-path `workspace.repo`
- bootstraps with `git worktree add`
- cleanup removes with `git worktree remove`

### `agent`

- runner: `name` (`pi` or `codex`)
- launch command: `command`
- model controls: `model`, `model_by_state`
- execution limits: `max_concurrent_agents`, `max_turns`
- runtime safety: `approval_policy`, `thread_sandbox`, `turn_sandbox_policy`
- timeouts: `stall_timeout_ms`, `read_timeout_ms`, `turn_timeout_ms` (runner-dependent)

### `prompts`

Per-state prompt injection:

```yaml
prompts:
  system: prompts/system.md
  repo: prompts/repo.md
  by_state:
    Todo: prompts/in-progress.md
    In Progress: prompts/in-progress.md
    Agent Review: prompts/agent-review.md
    Merging: prompts/merging.md
    Rework: prompts/rework.md
  default: prompts/in-progress.md
```
