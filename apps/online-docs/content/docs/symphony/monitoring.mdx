# Monitoring

Symphony provides real-time observability through a terminal dashboard, web dashboard, HTTP API, and structured logging.

## Terminal Dashboard (TUI)

Enabled by default. Shows a Ratatui-based live view with:

- Running sessions with turn count, token usage, last activity, and active model
- Throughput sparkline
- Color-coded status dots (green = tool_start, blue = tool_end, red = tool_error)
- Blocked issues with blocker identifiers
- Retry queue
- Completed issues

Disable with `--no-tui` to get JSON log output instead.

## Web Dashboard

Available at `http://localhost:<port>`. Auto-refreshes every 2 seconds.

Shows running sessions, token summary (input/output/total), retry queue, completed issues, polling stats, blocked issues, and a link to the Linear project.

## HTTP API

### `GET /api/v1/state`

Full orchestrator state as JSON: running sessions, retry queue, completed issues, token totals, polling diagnostics.

### `GET /api/v1/:issue_identifier`

Per-issue projection (e.g. `GET /api/v1/KAT-42`). Returns 404 if the issue is not running or in the retry queue.

### `POST /api/v1/refresh`

Trigger an immediate Linear poll. Requests are coalesced — multiple concurrent POSTs result in one actual refresh.

## Logging

Symphony emits structured JSON logs via `tracing`.

```bash
# Default
RUST_LOG=info symphony WORKFLOW.md

# Verbose
RUST_LOG=debug symphony WORKFLOW.md

# Trace symphony internals only
RUST_LOG=symphony=trace,info symphony WORKFLOW.md
```

### Log files

Use `--logs-root` for rotating log files:

```bash
symphony WORKFLOW.md --logs-root ~/symphony-logs
# Logs at: ~/symphony-logs/log/symphony.log
```

### Key log events

| Event | Description |
|-------|-------------|
| `notification_sent` | Slack notification delivered |
| `notification_failed` | Slack notification failed (warning) |
| `worker_stalled` | Agent session exceeded stall timeout |
| `worker_completed` | Agent session finished |
| `issue_state_changed` | Issue state changed during session |
| `blocked_by_dependency` | Issue has non-terminal blockers |
