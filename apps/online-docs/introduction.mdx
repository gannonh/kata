## Product overviews

### Kata CLI

Kata CLI is the backend IO and setup CLI used by Kata skills. It provides setup, environment diagnostics, and an operation interface (`kata call`, `kata json`) for backend-neutral workflow operations such as issues, tasks, statuses, artifacts, and health checks.

### Kata Desktop

Kata Desktop is an Electron app for working with Pi agent sessions across multiple workspaces. It includes streamed chat, workspace-scoped session history, permission modes, provider settings, workflow kanban, and Symphony controls.

### Kata Symphony

Kata Symphony is a headless orchestrator written in Rust. It polls Linear projects or GitHub Projects v2 boards for candidate issues, dispatches parallel agent sessions to work on them, and manages the full ticket lifecycle from `Todo` through implementation, PR creation, automated code review, human review, and merge.

Symphony runs on a server or in CI. Kata CLI provides setup, diagnostics, and backend operations (`kata setup`, `kata doctor`, `kata call`, `kata json`) used by Kata skills and Symphony workflows.

### Kata Context

Kata Context indexes source files using tree-sitter, builds a dependency graph stored in SQLite, and exposes commands for graph queries, grep, and file discovery. It provides structural, semantic, and memory-based codebase understanding to AI coding agents.
