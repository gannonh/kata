## Provider management

Desktop provides a UI for managing AI provider credentials in **Settings → Providers**. Credentials are stored in the shared Kata auth store at `~/.kata-cli/agent/auth.json`.

Provider records configured through Kata CLI are available in Desktop, and provider records configured through Desktop are available to Pi and Kata CLI through the same auth store.

## Integration configuration

Desktop runs Pi as a subprocess and uses Kata CLI for backend IO, so it uses shared Kata configuration files.

| Integration | Configure in | Used by Desktop |
|---|---|---|
| AI provider auth | `~/.kata-cli/agent/auth.json` or Settings | Yes |
| MCP servers | `~/.kata-cli/agent/mcp.json` | Yes, through Pi |
| Workflow preferences | `<workspace>/.kata/preferences.md` | Yes |
| Symphony project config | `<workspace>/.symphony/WORKFLOW.md` | Yes |
| Symphony connection URL | `<workspace>/.kata/preferences.md` or `KATA_SYMPHONY_URL` | Yes |

If you already use Kata CLI with MCP, Linear, or GitHub integrations configured, Desktop uses the same configuration where applicable.

## Setup flow

1. Configure providers in Desktop or through Kata CLI.
2. Add MCP servers in `~/.kata-cli/agent/mcp.json` when needed.
3. Define workspace workflow preferences in `.kata/preferences.md`.
4. Define Symphony config in `.symphony/WORKFLOW.md` when using managed Symphony.
5. Open the workspace in Desktop and start a session.

## Security

- Credentials are stored in the shared Kata auth store.
- Desktop masks credential values in Settings.
- Workspace files store backend and Symphony configuration, not provider secrets.
- Symphony runtime errors are surfaced without intentional secret exposure.
