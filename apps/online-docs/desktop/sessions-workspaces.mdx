## Sessions

Each chat in Kata Desktop is a **session** backed by a JSONL file. Sessions are stored in `~/.kata-cli/sessions/` and shared with Kata CLI.

The session sidebar supports:

- **New Session** to start a fresh conversation
- **Refresh** to reload the session list
- Session list sorted by most recently modified
- Automatic skipping of corrupted session files with warnings

## Workspaces

Use the workspace button in the top bar to switch directories. When you change workspace:

1. Desktop updates the working directory for the agent subprocess.
2. The session list re-scopes to sessions from that workspace.
3. Planning and workflow views refresh to match the new project.

Your most recent workspace is persisted so Desktop restores it on next launch.

## Persistence and shared state

| Item | Location | Notes |
|---|---|---|
| Session transcripts | `~/.kata-cli/sessions/*.jsonl` | Shared with Kata CLI |
| Auth credentials | `~/.kata-cli/agent/auth.json` | Provider keys used by Desktop and CLI |
| Desktop settings | Electron user data directory | UI-level preferences |
| Workflow preferences | `<workspace>/.kata/preferences.md` | Planning, workflow, and Symphony config |

## Permission modes

Desktop exposes three permission modes in the chat toolbar:

| Mode | Behavior |
|---|---|
| **Explore** | Read-only tool behavior |
| **Ask** | Prompt before write operations |
| **Auto** | Automatically approve operations |

## Thinking levels

For reasoning models (Anthropic Claude, OpenAI o-series), a thinking level toggle appears below the chat input:

**off · minimal · low · med · high · xhigh**

Higher levels give the model more reasoning budget. The `xhigh` option appears only for models that support it.

## Keyboard shortcuts

| Shortcut | Action |
|---|---|
| `Enter` | Send message |
| `Shift+Enter` | New line in input |
