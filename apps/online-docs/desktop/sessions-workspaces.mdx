## Sessions

Each chat in Kata Desktop is a session backed by a JSONL transcript. Desktop stores sessions in `~/.kata-cli/sessions/`, so the session history can be inspected by other Kata tooling.

The session sidebar supports:

- New sessions for fresh conversations
- Session switching with persisted history
- Session archiving and restore from Settings
- Refreshing the session list
- Session ordering by recent activity
- Corrupted session skipping with warnings

## Workspaces

Use the workspace picker in the top bar to switch directories. When you change workspace:

1. Desktop updates the working directory for the agent subprocess.
2. The session list re-scopes to sessions from that workspace.
3. Workflow board and Symphony views refresh against the selected project.

Your most recent workspace is persisted so Desktop restores it on next launch.

## Persistence and shared state

| Item | Location | Notes |
|---|---|---|
| Session transcripts | `~/.kata-cli/sessions/*.jsonl` | Shared transcript store |
| Auth credentials | `~/.kata-cli/agent/auth.json` | Provider credentials used by Pi and Kata CLI |
| Desktop settings | Electron user data directory | UI-level preferences |
| Workflow preferences | `<workspace>/.kata/preferences.md` | Backend and workflow settings |
| Symphony project config | `<workspace>/.symphony/WORKFLOW.md` | Managed Symphony launch config |

## Permission modes

Desktop exposes permission modes in the chat toolbar:

| Mode | Behavior |
|---|---|
| **Explore** | Read-oriented agent work |
| **Ask** | Prompt before write operations |
| **Auto** | Automatically approve supported operations |

## Thinking levels

For models with reasoning controls, a thinking level selector appears below the chat input:

`off · minimal · low · med · high · xhigh`

Higher levels give supported models more reasoning budget. The `xhigh` option appears only for models that support it.

## Keyboard shortcuts

| Shortcut | Action |
|---|---|
| `Enter` | Send message |
