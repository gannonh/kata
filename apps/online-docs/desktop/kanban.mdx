---
title: "Workflow kanban"
description: "How Kata Desktop renders Linear or GitHub workflow state in the right pane."
---

Kata Desktop uses the right pane for active workflow state. The kanban board shows milestone, slice, and task status from the configured backend.

## What the board shows

The board can display:

- Active milestone context
- Slice cards and child task summaries
- Workflow columns such as backlog, todo, in progress, review, and done
- Backend status for Linear or GitHub-backed workflow data
- Stale or error states when workflow data cannot refresh

## Symphony execution metadata

When Desktop is connected to Symphony, cards can include runtime metadata:

- Active worker assignment
- Current tool activity
- Pending escalation count
- Runtime connection indicators

This lets you correlate tracker state with active orchestration work.

## Manual view control

The right pane currently exposes two surfaces:

- **Kanban** for workflow state
- **Symphony** for worker activity, runtime health, and escalations

Use the pane controls to switch between them.

## Error and stale states

The board surfaces fetch and runtime failures explicitly:

- Workflow fetch failures show status and error metadata.
- Stale workflow data remains visible while refresh errors are reported.
- Symphony disconnects annotate runtime freshness without clearing workflow cards.
