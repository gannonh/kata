Kata Desktop includes a built-in Symphony operator surface in **Settings → Symphony**.

## Two operating modes

### Managed Symphony

Use the Start, Stop, and Restart controls in Desktop. Desktop resolves launch configuration from workspace preferences and environment, then runs Symphony as a managed subprocess.

The runtime panel shows the current lifecycle state:

- **Ready** — Symphony is running and the API is responding
- **Starting** / **Stopping** / **Restarting** — transition states
- **Stopped** — process terminated cleanly
- **Failed** — process crashed (exit code and stderr diagnostics are shown)
- **Config Error** — missing URL, workflow path, or binary (with actionable remediation text)

### External Symphony

If Symphony is already running outside Desktop, click **Dashboard Refresh** in Settings → Symphony. The dashboard connects to whatever is at the configured URL. The Runtime section tracks only the managed process, so it stays in its last state while the Dashboard section goes green independently.

## Live dashboard

The dashboard displays real-time operator signals:

- Worker rows: issue, state, tool, model, last activity
- Queue and completion counts
- Pending escalations
- Connection freshness (connected, reconnecting, disconnected)

## Escalation handling

1. A worker escalates with a question.
2. The escalation appears in Desktop with the issue identifier and question preview.
3. You submit a response inline.
4. Desktop sends the response to Symphony and the worker resumes.

## Configuration

Symphony configuration is workspace-driven. Set these fields in `<workspace>/.kata/preferences.md`:

```yaml
symphony:
  url: http://localhost:8080
  workflow_path: /path/to/WORKFLOW.md
```

For managed mode, the Symphony binary is resolved from:

1. `KATA_SYMPHONY_BIN_PATH` environment variable
2. `symphony` on `PATH`
3. Bundled binary in the packaged app (`Contents/Resources/symphony`)

If configuration is missing or invalid, Desktop surfaces a Config Error with specific remediation steps.

## Kanban integration

Symphony execution state is correlated into workflow kanban cards. Each card shows:

- Worker assignment badge and state
- Live tool activity indicator (e.g. "Execution: bash")
- Pending escalation count

See [Planning and workflow kanban](/desktop/planning-kanban) for details on the right-pane behavior.
