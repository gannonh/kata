## Provider management

Desktop provides a UI for managing AI provider credentials in **Settings → Providers**. Supported providers include Anthropic, OpenAI, Google, and Mistral.

Credentials are stored in the shared Kata auth store at `~/.kata-cli/agent/auth.json`, so providers configured in Desktop are available in Kata CLI and vice versa.

## Integration configuration

Because Desktop runs Kata CLI as a subprocess, it uses the same configuration files:

| Integration | Configure in | Used by Desktop |
|---|---|---|
| AI provider auth | `~/.kata-cli/agent/auth.json` (or Settings → Providers) | Yes |
| MCP servers | `~/.kata-cli/agent/mcp.json` | Yes (through CLI runtime) |
| Workflow preferences | `<workspace>/.kata/preferences.md` | Yes |
| Symphony connection | `<workspace>/.kata/preferences.md` | Yes |

If you already use Kata CLI with MCP, Linear, or GitHub integrations configured, Desktop inherits that configuration automatically.

## Setup flow

1. Configure providers in Desktop (**Settings → Providers**) or via Kata CLI.
2. Add MCP servers in `~/.kata-cli/agent/mcp.json` (see [CLI MCP support](/cli/mcp-support)).
3. Define workspace workflow preferences in `.kata/preferences.md`.
4. Open the workspace in Desktop and start a session.

## Security

- Credentials are stored in the shared Kata auth store, not in the workspace.
- Desktop shows masked credentials in the Settings UI.
- Symphony runtime errors are surfaced without exposing secret values.
