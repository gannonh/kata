# Project Artifact Template

Use this as the content shape for project-scoped `project-brief` artifacts.

```markdown
# [Project Name]

## What This Is

[Two or three sentences describing what the product does and who it is for. Use the user's language.]

## Core Value

[The one thing that matters most. This drives tradeoffs.]

## Target Users

- [User/persona/group]

## Requirements

### Validated

(None yet -- ship to validate)

### Active

- [ ] [Requirement hypothesis]

### Out of Scope

- [Excluded capability] -- [reason]

## Context

[Background, prior work, technical environment, domain knowledge, or constraints that planning should preserve.]

## Constraints

- **[Type]**: [What] -- [Why]

## Key Decisions

| Decision | Rationale | Outcome |
|---|---|---|
| [Choice] | [Why] | Pending |

## Open Questions

- [Question]

---
Last updated: [date] after project initialization
```

## Guidance

- Capture all durable decisions made during questioning.
- Keep "Core Value" sharp and singular.
- Use Active requirements as hypotheses until shipped and verified.
- Move invalidated scope to Out of Scope with a reason.
