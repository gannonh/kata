# S01: Rename and Strip — Research

**Date:** 2026-03-13

## Summary

This is a pure rename + file deletion slice. The project currently ships as `get-shit-done-cc` with a `get-shit-done-cc` bin and a `get-shit-done/` source directory. The rename touches ~110 files across agents, commands, hooks, workflows, templates, tests, docs, scripts, and package.json. No new logic is introduced — every change is a search-and-replace or file rename.

The rename has four distinct categories: (1) the top-level `get-shit-done/` directory → `kata/`, (2) the `commands/gsd/` directory → `commands/kata/`, (3) `agents/gsd-*.md` → `agents/kata-*.md`, (4) `hooks/gsd-*.js` → `hooks/kata-*.js` (minus `gsd-statusline.js` which is deleted outright). All internal string references (`gsd`, `get-shit-done`, `TÂCHES`) must be updated inside files too.

Tests currently pass (535 pass, 0 fail). The primary risk is forgetting a path reference that breaks test resolution or the bin at runtime. The safest execution strategy is: rename directories → rename files → sed-replace strings inside files → run `npm test` to confirm.

## Recommendation

Execute as a single sweep in one task:
1. Rename `get-shit-done/` → `kata/`
2. Rename `commands/gsd/` → `commands/kata/`
3. Rename each `agents/gsd-*` → `agents/kata-*`
4. Delete `hooks/gsd-statusline.js`; rename `hooks/gsd-check-update.js` → `hooks/kata-check-update.js`; rename `hooks/gsd-context-monitor.js` → `hooks/kata-context-monitor.js`
5. Update all string references in all files (sed or Node script)
6. Update `package.json` (name, bin, files, description, author)
7. Update `scripts/build-hooks.js` (remove statusline entry, rename hook filenames)
8. Run `npm test` — fix any failures
9. Run `npm pack` and grep tarball for `gsd`/`get-shit-done`/`TÂCHES` (excluding CHANGELOG)

A single-task approach avoids partial-state issues where tests fail mid-rename.

## Don't Hand-Roll

| Problem | Existing Solution | Why Use It |
|---------|------------------|------------|
| Mass string replacement | `find` + `sed -i` or Node `fs.readFileSync`/`writeFileSync` | Faster and more reliable than editing 110 files manually |
| Verifying packed tarball | `npm pack --dry-run` + `tar -tzf` + `grep` | Exact same check as milestone acceptance criteria |

## Constraints

- `npm test` must remain green throughout — tests resolve paths relative to the renamed `kata/` directory
- `hooks/dist/` is gitignored build output; `build:hooks` runs as `prepublishOnly` — update `scripts/build-hooks.js` so `npm run build:hooks` produces `hooks/dist/kata-check-update.js` and `hooks/dist/kata-context-monitor.js` (no statusline)
- `package.json` `files` array controls what `npm pack` includes — must update `get-shit-done` → `kata` and `commands/gsd` → `commands/kata` if commands are included
- `commands/` is not in `files` currently — verify whether it should be added; currently not shipped in tarball
- CHANGELOG.md is explicitly excluded from the gsd grep check per D005

## Common Pitfalls

- **Forgetting `commands/gsd/` in the files array** — `commands` is not in `package.json` `files`, so it is not packed. Either add it or leave it out; just be consistent.
- **Partial rename of `gsd-tools.cjs`** — The file is referenced by path in `tests/helpers.cjs` and by string in `bin/install.js`. Both must update together or tests break immediately.
- **`hooks/dist/` containing old filenames** — `hooks/dist/` is generated by `build:hooks` and included in the pack via `"hooks/dist"`. If `build-hooks.js` still writes `gsd-*` filenames the tarball will contain them. Delete `hooks/dist/` before rebuilding.
- **sed on binary/non-text files** — `assets/terminal.svg` and `.DS_Store` may match; confirm SVG text references and skip binary files.
- **Codex marker string in `bin/install.js`** — `Kata_CODEX_MARKER` is used to detect existing config blocks in users' `config.toml`; renaming it changes the marker for existing users. For a clean rebrand this is acceptable, but note it as a breaking change.
